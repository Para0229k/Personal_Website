製作動機

在學習C# Windows mode的過程中，我發現我有了圖片物件、讓物件之間互動和讓物件移動的方式(甚至形成動畫的效果)，我就開始想「那我好像已經有足 夠的程式能力能夠製作一款簡單的遊戲了，不如來試試看」，於是便產生了這支程式。

Chaos Stage的名字一部分是源自太空戰士中的歷代反派Chaos，我在取名字的時候剛好想到它，配合每關不同的特殊事件，就取出了"Chaos Stage"這個名 字，也讓玩家在看到這個名字的時候做好心理準備，面對隱藏在單純外表底下的混亂邪惡關卡。

遊戲流程

功能介紹

  1. 選擇關卡：在選擇關卡畫面點擊不同按鈕可以從不同關卡開始
  2. 角色移動：
    • 按左右移動、上跳躍(無法跑出地圖外)
    • 敵人會在一定範圍內巡邏
    • 角色和敵人圖片都真的會動(動畫效果)
  3. 角色發動攻擊：按空白鍵發射子彈攻擊敵人
  4. 碰撞判定：
    • 角色碰到障礙物或敵人會死亡
    • 敵人碰到子彈會死亡
    • 角色碰到冰棒(寶藏)就會過關
  5. 暫停遊戲：按暫停按鈕暫停遊戲
  6. 跳過關卡：真的受不了的話可以按跳過按鈕跳過當前關卡

額外功能

為了增加使用者的遊戲體驗，有一些額外的功能以加強遊戲的細節

  1. 重力模擬：
    • 角色掉落速度會隨著掉落時間變快，模擬重力
    • 走出地板外會自動掉落
  2. 畫面置中：不管視窗怎麼縮放畫面都還是在正中間
  3. 背景音樂：遊戲中會播放背景音樂，比較不無聊一點
  4. 事件音效：
    • 成功過關和闖關失敗會有不同的結算畫面、音效
    • 主要按鈕按下時也會有音效
  5. 操作顯示：右下角即時顯示使用者按鍵，讓旁觀者比較清楚某些關卡事件發生了什麼事

關卡設計

  1. 第一關：無事件，正常關卡
  2. 第二關：角色左右控制相反
  3. 第三關：怪物會高速反彈子彈，角色碰到怪物反彈的子彈會死亡
  4. 第四關：角色在跳過障礙的時候，障礙會往前移動
  5. 第五關：畫面會反覆變暗、變亮(5秒/次)，但只有變暗的時候可以移動
  6. 第六關：怪物變成三隻，其中兩隻無敵
  7. 第七關：共有三個寶物，只有一個是真的，另外兩個是陷阱(碰到會重新開始)
  8. 第八關：平台隱形，吃到寶物之後，會重新開始並出現下一個寶物，每次平台都會更換位置
  9. 第九關：所有平台消失，沒有隱形平台，要用滑鼠拖動角色上去
  10. 第十關：角色跳躍鍵失效，要用滑鼠點擊角色來跳躍

美術設計

皆使用免費公開的素材

  1. 背景圖片、音樂： Moonwind
  2. UI圖示： Kenny
  3. 角色、障礙物、敵人、子彈圖示： Dotown
  4. 按鈕音效： 効果音ラボ

遇到的挑戰

  1. 如何在自動落下和跳躍的過程中判定角色落地(太嚴格會導致來不及判定就穿過去)
  2. 如何切換關卡
  3. 切換關卡後物件消失
  4. 對UI配置、美術設計較不熟悉

解決方法

  1. 將垂直移動寫在Timer中，並把移動距離拆成每次1px以避免來不及判定
  2. 活用Panel等容器，只轉移每關共通物件
  3. 善用debug mode來觀察原因(座標定位跑掉)
  4. 尋找現成素材，搭配UX的概念，以簡單明瞭為主

得到的收穫

  1. 對C#語言的掌握度提升
  2. 對物件之間、物件和使用者間的互動情形有更多的瞭解
  3. 體驗一個完整的遊戲的製作流程

製作動機

在學習工程數學的Fourier transform的過程中，我對於Shazam這個應用程式的演算法感到好奇，但由於當時我沒有足夠的程式能力支援我去研究它，好奇與 疑問就這樣被保存在心裡。

直到我掌握了MATLAB語言後，我發現我有足夠的基礎去支持我往音訊分析的方面研究了，音訊分析的第一步就是對訊號進行視覺化、頻譜分析及濾波等操 作，我便以此為題，製作一個MATLAB應用程式。

UI介紹

  1. 展示音訊視覺化結果
  2. 選擇圖像種類
  3. 選擇濾波器種類
  4. 客製化濾波器種類
  5. 音訊振幅及速度(採樣率)調整
  6. 音訊相關資訊展示
  7. 當前播放狀態指示燈
  8. 播放(調整後的)音訊
  9. 從電腦中讀取音訊檔案
  10. 透過麥克風錄製新檔案
  11. 儲存調整後的音訊檔案
  12. 客製化濾波器參數
  13. UI主題

功能介紹

  1. 音訊視覺化：可顯示波形圖、頻譜圖、時頻譜，並隨時更改標題
  2. 音訊過濾：包含內建的低通、高通、帶通、帶阻FIR濾波器及自訂參數的濾波器(含輸入防呆)
  3. 音訊編輯：可使用拉桿調整音訊的音量和速度
  4. 音訊播放：播放編輯後的音訊，附播放狀態指示燈
  5. 音訊管理：可讀取音訊(.wav檔)、錄製音訊、儲存編輯後的音訊
  6. 顏色主題：Dark mode / Light mode主題切換，保護眼睛

遇到的挑戰

  1. 對訊號的基本原理不夠熟悉
  2. 如何在MATLAB中操作訊號
  3. 對濾波器的種類及運作方式不夠熟悉
  4. 如何針對客製化參數進行輸入防呆

解決方法

  1. 自行透過網路資源進行學習
  2. 查看MathWorks的官方說明檔案
  3. 自行透過網路資源進行學習
  4. 針對每一種可能的情況進行強制修正

得到的收穫

完成這個專案後，我對訊號本身有了基本的理解，也瞭解了如何透過程式碼來對訊號進行操作，更重要的是，現在我在寫程式的過程中可以透過官方說明 文件、Stack Overflow等網路管道來解決我面臨的問題了。

網站導覽

  1. 首頁：分成歡迎、分頁介紹、畫廊、聯絡方式四個部分，主要是介紹我的個人網站

  3. 作品集：分成展示區、作品一覽和詳細資料三個區塊，介紹我的作品及製作歷程

特點介紹

  1. 所有的card物件在滑鼠停留時會浮起來

  2. 查看Bootstrap, MDN, W3School等網站的說明文件
  3. 命名需要命的更清楚且不重複
  4. 由於現在的網頁檔案不大，先以能夠表現出想要的效果為主，未來多做多練習以後自然會進步
5. 繼續努力吧!

製作動機

上完 "Introduction to data, signal and image analysis with MATLAB" 這門課以後，我想將所學轉換成更具應用價值的形式，為了往電腦視覺的方向前進， 我想先嘗試製作一個用於辨識圖像的應用程式，正好課堂作業有寫出了許多可用的函式，我只需要將其改寫並使用就好。

我先是製作了一個用於偵測內建圖像的腳本，但我想讓它變得更泛用一點，不是重複偵測相同的圖案，於是我將其拓展成應用程式，將視窗應用程式、圖像處 理、機器學習融合在一個專案當中。

UI介紹

  1. 展示原影像
  2. 展示經過處理後的影像
  3. 選擇要偵測的幣值
  4. 切換右側的影像種類
  5. 從電腦中讀取影像
  6. 偵測結果

功能介紹

  1. 影像切換：可切換察看經過Otsu threshold、膨脹、侵蝕的圖像，或是最終結果
  2. 幣值選擇：可以選擇要偵測的幣別，增加泛用度
  3. 讀取影像：可從電腦中讀取要被偵測的影像，增加泛用度
備註：經過調整參數後，這是目前的平均最佳表現，仍然有1~2個硬幣會被判別錯誤。

遇到的挑戰

  1. 對影像形態學處理的不熟悉
  2. 如何在泛用度與準確度之間做權衡
  3. 如何將不同的圖片的大小、解析度統一
  4. 如何確定前後景的亮暗關係
  5. 如何做得更好?(解決拍攝距離造成硬幣大小規格不同的情形)

解決方法

  1. 查看MathWorks的官方說明檔案，瞭解如何在MATLAB上執行侵蝕、膨脹等型態學處理
  2. 透過Grid Search，找到讓測試圖片中的平均誤差降到最低的參數
  3. 活用課堂所學，解決解析度造成的判別誤差
  4. 將圖片先透過一次Otsu thoreshold抓出前景影像，在將其與四個角落的平均亮度做比較
  5. 尚未解決，可能需要有能夠自動設定不同種類硬幣直徑的功能，我會繼續學習直到我可以做出這個功能再來改進這支程式

得到的收穫

在製作這個專題的過程中，我得以將我目前為止學到的幾種大領域的知識結合在一起，讓他們在我的腦海中不再只是單獨的分支，而是交錯在一起的知識網 路。隨著越來越多新知識的引入，我也感受到這個「知識連通」的作用有多重要，這樣的專題可以幫助我梳理並複習我的所學，在製作的過程中也會突然發 現「原來這兩個東西是有關連的!」，每當出現這樣的新理解都能讓我不斷成長、加深我對這些領域的瞭解。

製作動機

重拾程式後，我透過Stanford University的開放式課程Code in Place學習了基本的Python語法，雖然對課堂所學和作業中片段的小程式得心應手，但我時常 想起老師們常說的話：「真正的程式沒有這麼簡單，從幾百、幾千，甚至幾萬行的都很常見」，於是不滿足於課堂作業的我想要看看真正的程式是什麼樣子的。

於是我選擇了俄羅斯方塊這個主題，因為這個經典的小遊戲邏輯簡單、清楚，又有一定的深度和許多種變化，很適合拿來當作臨摹學習的主題。

遊戲流程

功能介紹

  1. 方塊操作：基本的加速落下、旋轉方塊等功能
  2. 額外分數：單次消除的行數越多，額外給分越多
  3. 關卡等級：每10000分上升一級並加快速度
  4. 預覽方塊：可以在左上角預覽下一個預計掉落的方塊
  5. Holding：按空白鍵可以保留落下中的方塊
  6. 除錯模式：按D可以顯示程式運作情形(用數字表示方塊)

遇到的挑戰

  1. 非常多第一次看到的語法及函式
  2. 方塊進行不合理的旋轉後造成覆蓋或消失
  3. 方塊堆滿一列後沒有被清除
  4. 如何提升方塊的下落速度
  5. 如何在現有基礎上增加新功能

解決方法

製作動機

JavaScript是基本前端網頁開發的最後一塊拼圖，學到這裡我覺得我應該已經能夠開發出一個互動式的網站了，於是先選擇了桌遊這種單純、簡單的主題作為 練習，之所以會想要選擇桌遊作為網頁遊戲的首個專題，也是因為我之後想要製作可以多人連線的網頁遊戲，在我學好完整的網頁開發之前，先將此題目當作 一個前哨戰，考驗我的前端架構能力。

遊戲流程

遊戲本身比實體的Simon難一點，原則是相同的，就是"Do what Simon says"，但在這個網頁中每次只會顯示下一步，並不會重複前面的順序，藉此來提高 難度，讓遊戲更具挑戰性。

功能介紹

  1. 按鍵音效：四個按鍵分別有各自的音效，在Simon出題時和玩家按下時都會播放
  2. 按鍵回饋：按下按鍵後會有變暗的動畫，讓使用者能知道按下哪顆按鍵
  3. 遊戲結束：遊戲結束時會播放音效、並藉由鍵盤重新開始

遇到的挑戰

  1. 如何記錄每一次的指令
  2. 如何將玩家的輸入按鍵與題目指令對照
  3. 如何製作出「按下按鈕」的特效

解決方法

  1. 善用Array，並用.push()函式紀錄每次的指令
  2. 每當使用者按下按鍵，就紀錄該按鍵種類及按的次數，並檢查是否符合對應的題目指令
  3. 多去別的網站看看按下按鈕應該要有什麼效果

得到的收穫

經過這個專案，我對於CSS的設計、JavaScript的互動都有了更深刻的理解，在逛各個網站的過程中也更能夠注意他們的設計、排版、功能、配色及背後的用 意並加以學習，這個學習網頁設計的過程也讓我有和學程式一樣的感覺，兩者都是透過經驗來累積自身實力，所以多看、多想、多做的過程本身都在讓我變得 更強，總而言之，這個簡單的網頁遊戲除了讓我更熟練掌握前端開發的互動以外，還讓我更清楚學習網頁設計的方式。
